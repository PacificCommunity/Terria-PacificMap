Region Mapping

How to...

What is region mapping?

Region mapping is a technique to display data which has a geographic relationship, but does not contain the geographic definition.

For example, if we have a spreadsheet containing population data of countries. The spreadsheet doesn't contain the geographic boundaries, but because they are already known to PacificMap we can display the population data on the a map.

What is csv-geo-pacific?

csv-geo-pacific is just the formal name given to the region mapping data specification that PacificMap uses. It defines what layers are available and how users can align their data to those layers.

How do I use it?

The key thing for working with region mapping is having a well-structured csv file. To be able to visualise the data the csv must contain a column header which PacificMap and the csv-geo-pacific specification understand.

For example if you had population data for Districts in American Samoa you could include the ASM_DISTRICT_CODE column header to display it on PacificMap.

ASM_DISTRICT_CODE District Population
ASM-1 Eastern 23,441
ASM-2 Manu'a 1,143
ASM-3 Unorganized 17
ASM-4 Western 32,435

To visualise the data you can then drag-and-drop the CSV onto PacificMap.

PacificMap will warn you if you if it it unable to visualise your CSV.

So far we've configured PacificMap to support to the following region data

  • Exclusive economic zone (eez) boundaries
  • Sub-Country boundaries Level 1
  • Sub-Country boundaries Level 2
  • Sub-Country boundaries Level 3

Exclusive Economic Zone boundaries

What boundaries are included?

The Exclusive Economic Zone boundary set provides a layer showing countries expanded by 200 nautical miles from their coast. This layer is helpful for representing countries in the Pacific which often have a small landmass.

What codes can I use?

  • EEZ's can be mapped using a country code
    • One of the following headers is required: co_id, CO_ID, COUNTRY_ID, SPC_COUNTRY_ID, SPC_AREA
    • Valid values for the codes are ISO2 country codes, eg "AS" for American Samoa
  • EEZ's can be mapped using a country name.
    • One of the following headers is required: coname, CO_NAME, country_name, COUNTRY_NAME, COUNTRY, Countries, SPC_COUNTRY, SPC_COUNTRY_NAME
    • Using country names is a less prefered option compared to using country codes as they are trickier to get align consistently to the specification.

How do I map my data to these regions?

This layer can be used by including one of the column headings specific below in your data. There are a number of options depending on your preference.

  • SC1_CODE for non-country-specific use, eg you'd like to display data from multiple countries in a single csv.
  • {ISO3}_SC1_CODE eg ASM_SC1_CODE for Sub-Country 1 Codes in American Samoa.
  • {ISO3}_{LOCAL ADMIN TYPE}_CODE eg ASM_DISTRICT_CODE for Districts in American Samoa.
  • Using the patterns above valid column headers are:
    SC1_CODE, ASM_SC1_CODE, ASM_DISTRICT_CODE, COK_SC1_CODE, COK_GROUP_OF_ISLANDS_CODE, FJI_SC1_CODE, FJI_DIVISION_CODE, PYF_SC1_CODE, PYF_ADMINISTRATIVE_SUBDIVISIONS_CODE, FSM_SC1_CODE, FSM_STATE_CODE, NRU_SC1_CODE, NRU_ISLAND_CODE, NCL_SC1_CODE, NCL_PROVINCE_CODE, MNP_SC1_CODE, MNP_ISLAND_CODE, PNG_SC1_CODE, PNG_PROVINCE_CODE, SLB_SC1_CODE, SLB_PROVINCE_CODE, TON_SC1_CODE, TON_DIVISION_CODE, TUV_SC1_CODE, TUV_ISLAND_CODE, VUT_SC1_CODE, VUT_PROVINCE_CODE, WLF_SC1_CODE, WLF_DISTRICT_CODE
  • Note: We don't provide any option to map administrative boundaries using names as they as they sometimes are not unique across or within countries.
